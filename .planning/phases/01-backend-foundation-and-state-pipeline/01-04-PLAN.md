    1. Write src/ws/messages.rs -- WebSocket message types:
       - Server-to-client (WsMessage, serde tagged enum with #[serde(tag = "type")]):
         - "snapshot": { project: String, data: ProjectState } -- full state for one project
         - "delta": { project: String, changes: Vec<StateChange> } -- incremental updates
         - "health": { uptime_secs: u64, db_size_bytes: u64, ws_client_count: u32, watcher_queue_depth: u32, memory_usage_bytes: u64, per_project_status: HashMap<String, ProjectWatcherStatus> }
         - "error": { message: String, code: String }
       - Client-to-server (ClientMessage, serde tagged enum):
         - "subscribe": { projects: Vec<String> }
         - "unsubscribe": { projects: Vec<String> }
       - ProjectState struct: project (ProjectRow), phases (Vec<PhaseStateRow>), plans (HashMap<String, Vec<PlanStateRow>>), recent_runs (Vec<ExecutionRunRow>), agent_sessions (Vec<AgentSessionRow>), verifications (HashMap<String, VerificationResultRow>), config (Option<GsdConfig>), parse_errors (Vec<ParseErrorRow>)
       - ProjectWatcherStatus: { active: bool, watched_paths: u32, last_event_at: Option<String>, error_count: u32 }

    2. Write src/broadcast.rs -- Per-project broadcast management:
       - Broadcaster struct containing:
         - channels: RwLock<HashMap<String, broadcast::Sender<StateUpdate>>> (project_id -> sender)
         - client_count: AtomicU32
       - Methods:
         - new() -> Self
         - get_or_create_channel(&self, project_id: &str) -> broadcast::Sender<StateUpdate>
         - subscribe(&self, project_ids: &[String]) -> Vec<(String, broadcast::Receiver<StateUpdate>)>
           - Returns receivers for each subscribed project
           - Increments client_count
         - unsubscribe(&self): decrements client_count
         - broadcast(&self, project_id: &str, update: StateUpdate): sends to project's channel
         - client_count(&self) -> u32

    3. Write src/ws/mod.rs -- WebSocket handler:
       - `pub async fn ws_handler(ws: WebSocketUpgrade, State(state): State<Arc<AppState>>) -> Response`
       - On upgrade, call handle_socket
       - handle_socket flow:
         a. Wait for first message -- must be Subscribe { projects }
         b. For each subscribed project: query DB for full ProjectState, send as Snapshot message
         c. Subscribe to broadcast channels for those projects
         d. Spawn send task: forwards broadcast messages as Delta JSON text frames
         e. Spawn receive task: handles Unsubscribe messages, ping/pong
         f. Spawn health heartbeat task: sends Health message every 7 seconds (midpoint of user's 5-10s range)
         g. tokio::select! on all tasks -- if any ends, abort others
       - Health heartbeat collects: uptime from AppState.start_time, db_size from PRAGMA page_count * page_size, ws_client_count from Broadcaster, watcher_queue_depth (from a counter in pipeline)
       - Handle send errors gracefully (client disconnect)

    4. Update src/state.rs AppState to include:
       - broadcaster: Broadcaster (from broadcast.rs)
       - Ensure broadcast_tx from pipeline is wired to Broadcaster

    5. Update src/main.rs:
       - Add WebSocket route: `.route("/api/v1/ws/state", get(ws::ws_handler))`
       - Wire Broadcaster into AppState
       - Spawn a task that subscribes to the pipeline's broadcast::Receiver<StateUpdate> and forwards to Broadcaster.broadcast() for the appropriate project
