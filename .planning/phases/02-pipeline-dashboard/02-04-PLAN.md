         - Looks up the project to get its filesystem path
         - Writes the config JSON to `{project_path}/.planning/config.json`

         - Error handling: 404 if project not found, 500 if write fails
       - `router()` function returning `Router::new().route("/config", put(update_config))`

    4. Create `AgentDropdown.svelte`:

       - Dropdown menu with agent options:

    5. Create `AgentRoutingPanel.svelte`:
       - Props: `phaseNumber: string`, `plans: PlanState[]`, `projectId: string`
       - Renders as a collapsible section within the phase detail view (collapsed by default with "Agent Routing" header)
       - When expanded, shows three tiers:
         a. **Project Default**: single AgentDropdown showing the project-level default agent
         b. **Stage Overrides**: 4 rows (Discuss, Plan, Execute, Verify) each with an AgentDropdown. Shows "Inherit default" if no override exists. Effective agent shown next to the dropdown.
         c. **Per-Plan Overrides**: one row per plan in this phase, each with plan number/name and AgentDropdown. Shows effective agent (resolved from cascade) with visual distinction (bold) if overridden.
       - Cascade resolution logic:
         - Effective agent for a plan = plan_override ?? stage_override[stage] ?? project_default ?? "claude"
         - Display the resolved agent as AgentBadge next to each dropdown
         - Overridden values (not inherited) shown with a bold label or small "overridden" indicator
       - On any dropdown change:
         - Build the updated config object
         - Send PUT /api/v1/projects/{projectId}/config with the full config JSON
         - Show toast via svelte-5-french-toast: "Agent routing updated"
         - No explicit save button -- changes auto-save immediately per user decision
       - Parse config from `projectStore.state.config?.config_json` (JSON string -> object)
       - If config is null, start with sensible defaults (project default = "claude", no overrides)
