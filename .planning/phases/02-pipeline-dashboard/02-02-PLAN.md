    1. Create `src/lib/utils/phase-sort.ts`:
       - `sortPhaseNumbers(phases: { phase_number: string }[])` -- sorts by parseFloat(phase_number) so "2.1" falls between "2" and "3", and "10" sorts after "9"
       - `isDecimalPhase(phaseNumber: string): boolean` -- returns true if phase_number contains a dot

    2. Create `src/lib/utils/duration.ts`:
       - `formatDuration(minutes: number | null): string` -- null returns "--", <1 returns "<1m", otherwise "Xm" or "Xh Ym"
       - `formatTimestamp(isoString: string | null): string` -- null returns "--", otherwise returns relative time ("2m ago", "1h ago", "Mar 6")
       - `calculatePhaseDuration(runs: ExecutionRun[]): number | null` -- sums duration_minutes from all non-superseded runs in a phase

    3. Create `PhaseChip.svelte`:
       - Props: `phase: PhaseState`, `runs: ExecutionRun[]`, `selected: boolean`, `density: 'rich' | 'medium' | 'minimal'`, `onclick: () => void`
       - Rich density (default): shows phase number, name, stage badge (colored pill), progress bar (completed/total plans), and duration
       - Medium density: phase number, name, stage badge only
       - Minimal density: phase number and colored dot for stage
       - Stage badge colors: map stage to status colors (planned->pending, executing->working, executed->done, verified->done)
       - Progress bar uses ProgressBar component: `(completed_plan_count / plan_count) * 100`
       - Duration: calculated from runs prop via `calculatePhaseDuration()`
       - Selected state: --bg-elevated background, --border-focus border
       - Unselected: --bg-surface background, --border-subtle border
       - Hover: subtle brightness increase
       - Click fires onclick callback

    4. Create `MilestoneGroup.svelte`:
       - Props: `milestone: string`, `phases: PhaseState[]`, `collapsed: boolean`, `onToggle: () => void`, `onSelectPhase: (phase: PhaseState) => void`
       - When collapsed: shows a single summary chip "v1.0 -- X phases" with checkmark if all complete
       - When expanded: renders individual PhaseChip components for each phase
       - Click on collapsed chip calls onToggle to expand
       - Completed milestones default to collapsed per user decision; current milestone always expanded

    5. Create `PhaseTimeline.svelte`:
       - Reads phases from `projectStore.state.phases`
       - Sorts phases using `sortPhaseNumbers()`
       - Groups phases by milestone (from project-level data; for now treat all as same milestone since per-phase milestone is not in PhaseState)
       - Renders MilestoneGroup components horizontally in a scrollable container
       - Horizontal scroll with overflow-x: auto, subtle scroll indicators (gradient fade on edges when overflowing)
       - Auto-selects the currently executing phase on initial render (stage === 'executing'); if none, selects next pending; if all done, selects last
       - Tracks `selectedPhaseNumber` as local $state, dispatches selection via callback prop `onPhaseSelect: (phaseNumber: string) => void`
       - Timeline strip stays fixed at top (sticky position) per user decision
       - --bg-surface background, bottom --border-subtle border
       - Shows skeleton loading state (3-4 skeleton chips) when projectStore.loading is true
