From src/ws/messages.rs:
```rust
pub enum WsMessage {
    Snapshot { project: String, data: ProjectState },
    Delta { project: String, changes: Vec<StateChange> },
    Health { uptime_secs: u64, db_size_bytes: i64, ws_client_count: u32, watcher_queue_depth: u32, memory_usage_bytes: u64, per_project_status: HashMap<String, ProjectWatcherStatus> },
    Error { message: String, code: String },
}

pub enum ClientMessage {
    Subscribe { projects: Vec<String> },
    Unsubscribe { projects: Vec<String> },
}

pub struct ProjectState {
    pub project: Project,
    pub phases: Vec<PhaseState>,
    pub plans: HashMap<String, Vec<PlanState>>,
    pub recent_runs: Vec<ExecutionRun>,
    pub agent_sessions: Vec<AgentSession>,
    pub verifications: HashMap<String, VerificationResult>,
    pub config: Option<ProjectConfig>,
    pub parse_errors: Vec<ParseError>,
}
```

From src/watcher/pipeline.rs:
```rust
pub enum StateChange {
    PhaseUpdated { phase_number: String, stage: String },
    PlanUpdated { phase_number: String, plan_number: String, status: String },
    VerificationUpdated { phase_number: String, status: String },
    ConfigUpdated,
    AgentHistoryUpdated { session_count: usize },
    ProjectStateUpdated { status: Option<String> },
    ParseError { file_path: String, error: String },
}
```

REST endpoints:
- GET /api/v1/projects -> Project[]
- GET /api/v1/projects/:id/state -> ProjectState
- GET /api/v1/projects/:id/files/*path -> raw file content
- GET /api/v1/health -> health diagnostics
- WebSocket /api/v1/ws/state -> subscribe/snapshot/delta protocol

  <action>
    1. Create `src/lib/stores/websocket.svelte.ts` as a class-based Svelte 5 rune store:
       - `WebSocketManager` class with `status = $state<ConnectionStatus>('disconnected')` where ConnectionStatus = 'connecting' | 'connected' | 'reconnecting' | 'disconnected'
       - `connect(projectIds: string[])` method: opens WebSocket to `${protocol}//${location.host}/api/v1/ws/state`
       - On `onopen`: set status to 'connected', reset reconnect delay to 1000ms, immediately send Subscribe message with project IDs (CRITICAL: must send within 10 seconds per backend timeout)
       - On `onmessage`: parse JSON, route to project store's `handleMessage()` based on message type
       - On `onclose`: set status to 'reconnecting', schedule reconnect with exponential backoff (1s -> 2s -> 4s -> ... -> 30s max)
       - On `onerror`: log error, let onclose handle reconnection
       - `disconnect()` method: clear reconnect timer, close WebSocket, set status to 'disconnected'
       - `send(msg: ClientMessage)` method for sending subscribe/unsubscribe
       - Export singleton: `export const wsManager = new WebSocketManager()`

    2. Create `src/lib/stores/project.svelte.ts` as a class-based rune store:
       - `ProjectStore` class with:
         - `state = $state<ProjectState | null>(null)` -- full reactive project state
         - `loading = $state(true)` -- true until first snapshot received
         - `projects = $state<Project[]>([])` -- project list for sidebar
         - `selectedProjectId = $state<string | null>(null)` -- currently selected project
       - `applySnapshot(data: ProjectState)` -- replaces full state, sets loading=false
       - `applyDelta(changes: StateChange[])` -- applies each change surgically:
         - PhaseUpdated: find phase by phase_number, update stage field
         - PlanUpdated: find plan in plans[phase_number] by plan_number, update status
         - VerificationUpdated: update verifications[phase_number]
         - ConfigUpdated: trigger config reload via REST (delta doesn't include data)
         - AgentHistoryUpdated: trigger agent sessions reload via REST
         - ProjectStateUpdated: update project.status
         - ParseError: add to parse_errors array
       - `fetchProjects()` -- GET /api/v1/projects, populates projects array
       - `selectProject(id: string)` -- sets selectedProjectId, triggers WebSocket subscribe
       - `handleMessage(msg: WsMessage)` -- routes snapshot/delta/health/error to appropriate handler
       - Export singleton: `export const projectStore = new ProjectStore()`

    3. Create `src/lib/stores/settings.svelte.ts` as a localStorage-backed settings store:
       - `SettingsStore` class with $state fields:
         - `outputLineCount = $state(10)` -- configurable output lines in expanded cards
         - `defaultCardState = $state<'collapsed' | 'expanded'>('collapsed')`
         - `autoExpandActive = $state(true)` -- auto-expand currently active plan card
         - `visibleStats = $state({ steps: true, commits: true, duration: true, wave: true })`
         - `autoScroll = $state(true)` -- auto-scroll output stream
         - `timelineDensity = $state<'rich' | 'medium' | 'minimal'>('rich')`
         - `fontSize = $state(14)` -- in pixels
         - `notifications = $state({ planCompletion: true, errors: true, phaseTransitions: true, agentSwitches: true })`
       - Constructor loads from `localStorage.getItem('gsdui-settings')`
       - `$effect` that persists to localStorage on any change (JSON.stringify the settings)
       - Export singleton: `export const settingsStore = new SettingsStore()`
  </action>
